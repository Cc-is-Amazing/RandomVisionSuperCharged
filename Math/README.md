# Math 模组说明

更新时间：2026-03-14

## 模组定位

Math 是一个“战斗与信息提示增强”模组，目标是在不改动原版流程的前提下，直接把推演结果显示在当前界面上，减少手算负担。

## 当前能力概览

- 战斗中显示即将承受的伤害。
- 在手牌上给出推荐的防御组合与斩杀组合高亮。
- 对可斩杀敌人显示额外标记，并闪烁血条。
- 提供抽牌概率、地图问号概率、药水空位掉率、商店购买收益等额外信息提示。

## 战斗提示规则

- 战斗提示文本已接入本地化，会随游戏语言切换刷新。

### 防御提示

- 只在当前战斗房间、且轮到玩家操作时显示。
- 红字显示这一轮预计会掉多少生命。
- 绿色说明文本会显示防御方案：
  - 能完全防住时：`1费格挡10，1费击杀酸液虫免伤2，溢出0`
  - 无法完全防住时：`1费虚弱咬人花免伤4，1费格挡4，缺少2`
- 会按来源拆开显示防御贡献，而不是只显示一个总防御值。
- 对 `虚弱免伤` 和 `击杀免伤`，文本会带上对应敌人名字，避免多敌人时看不出应该打谁。
- 防御说明使用富文本显示；其中敌人名字会额外用红色标出，方便快速定位目标。
- 这里的 `防N` 不只计算直接获得的格挡，也会把“击杀敌人减少的攻击”“上弱化后减少的伤害”等等折算进去，因此更贴近实际少吃到的伤害。

### 敌人头顶提示

- 敌人头顶会显示当前手牌对该目标的最佳攻击方案：未斩杀时为 `最多X费打N，剩余Y`，能斩杀时为 `最多X费打N，溢出Y`。
- 如果该方案依赖 `易伤`、`中毒` 等 debuff，也会在末尾标记，例如 `（含易伤）`。
- 只有真正可斩杀的方案才会驱动手牌红色高亮与敌人血条闪烁；普通“最多能打多少”的方案只用于文本提示。
- 头顶攻击提示使用固定宽度自动换行，并且会按自身高度向上延伸，减少英文长句遮挡或串到相邻敌人的情况。

### 卡牌高亮

- 防御方案中的卡牌使用绿色高亮。
- 斩杀方案中的卡牌使用红色高亮。
- 如果同一张牌同时被防御方案和斩杀方案选中，红色优先，避免攻击牌被误判成防御牌。

## 设计边界

- 这些提示属于“信息覆盖层”，不会替玩家自动出牌。
- 防御搜索会优先找能完全防住的低成本方案；如果完全防不住，则退化为当前搜索范围内“减伤最多”的方案。
- 所有结果都基于当前可见战斗状态与预览值做近似推演，若原版存在特殊时序或隐藏随机性，显示结果仍可能与最终结算略有差异。
